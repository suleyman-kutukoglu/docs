title: "Windows VM Oluşturma ve Erişim"
---

Zadara Edge Cloud üzerinde **mevcut bir Windows imajından** VM oluşturmak ve erişim sağlamak için aşağıdaki adımları izleyin.

## 1) VM Oluşturma Sihirbazını Açın

1. **Compute → Instances** bölümüne gidin.
2. **Create** butonuna tıklayın. Açılan sihirbaz dört sekmeden oluşur: **Compute**, **Storage**, **Networking**, **Config**.

---

![create-vm.png](/images/create-vm.png)

## 2) Compute Sekmesi

- **Name:** VM için bir ad girin.
- **Create From:** **Image** seçin.
- **Image:** Listeden Windows imajını seçin (ör. _Public - Windows Server 2025 Standard_).
- **Instance Type:** CPU / RAM kaynak profilini seçin.
- **Key Pair:** Windows için Administrator parolasını çözebilmek amacıyla bir **RSA (PEM)** key pair ilişkilendirin. Yeni bir anahtar oluşturmak için **\+** butonunu kullanabilirsiniz.
- **Opsiyonel İşaretler:**
  - **Power Up** — Sunucu oluşturulur oluşturulmaz otomatik başlatılsın. (Seçili ise VM sihirbaz tamamlandıktan sonra otomatik olarak başlar.)
  - **High Availability** — Fiziksel arıza vb. gibi durumlarda VM’in farklı bir fiziksel host üzerinde yeniden başlatılmasını sağlar.
  - **Protect from deletion** — VM'in sadece Tenant Admin rollerine sahip kullanıcılar tarafından silinmesine izin verir.
  - **Create Multiple** — Aynı konfigürasyona sahip birden fazla VM oluşturmak için kullanılır.
- **Next** ile devam edin.

## 3) Storage Sekmesi

- **Boot Volume:** Boot diskinin boyutunu ve tipini belirleyin.
- **Data Volumes (İsteğe bağlı):** İhtiyaç halinde ek veri diskleri ekleyin.
- **Next** ile ilerleyin.

![windows-vm-storage.png](/images/windows-vm-storage.png)

## 4) Networking Sekmesi

> Bu adımda **en az bir ağ arayüzü (NIC)** tanımlanması zorunludur. Varsayılan olarak bir tane seçilir gelir ek işlem yapmaya gerek yoktur.

- **VPC Seçimi:** Mevcut VPC’lerden birini seçin. (Varsayılan VPC ile devam edebilirsiniz.)
- **Subnet Seçimi:** Seçtiğiniz VPC içindeki bir subnet’i seçin. (Varsayılan subnet kullanılabilir.)
- **IP Ataması:**
  - **Automatic (DHCP)** — Sunucu otomatik IP alır (varsayılan).
  - **Static (Manuel)** — İsterseniz statik IP atayabilirsiniz.
- **Security Group:** Uygun security group’u seçin veya varsayılanı bırakın. Daha sonrasında gerekli konfigürasyonu yapacağız.
- **DNS Name:**
  - Aynı ağ (aynı VPC/subnet) içindeki sunucuların birbirlerini isimle bulabilmesi için kısa bir DNS adı girin (ör. `my-windows`).
  - Bu değer internal DNS kaydı olarak kullanılacak şekilde atanır ve ağ içi keşfi kolaylaştırır.
- **Next** ile devam edin.

![Networking sekmesi — Subnet, IP, Security Group](./images/windows/networking-tab.png)

- Opsiyonel olarak metadata veya etiketler ekleyebilirsiniz.
- **Finish** ile sihirbazı tamamlayın.

![Config sekmesi — opsiyonel ayarlar](./images/windows/config-tab.png)

## 6) VM’i Başlatma & Elastic IP

- Eğer **Power Up** seçili değilse, oluşturulan VM’i listeden seçip **Start** ile başlatın.
  - Eğer **Power Up** seçiliyse, sihirbaz tamamlandığında VM otomatik olarak başlatılır.
- Durum **Active** olduğunda VM erişime hazırdır.
- Dış ağdan (internet) erişim gerekiyorsa Elastic IP atayın: **Instances → (VM detayı) → Elastic IPs → Attach**.

![Elastic IP ekleme](./images/windows/attach-elastic-ip.png)

---

## 7) Windows Parolasını Çözme (Get Password)

1. **Compute → Instances** listesinden ilgili Windows VM satırına gelin ve **… More → Get Password** seçeneğini tıklayın.
2. Size sorulan alana VM ile ilişkilendirdiğiniz **private key (.pem)** dosyasının içeriğini yapıştırın veya dosyayı yükleyin.
3. **Retrieve** sonrasında ekranda **Administrator** parolasını göreceksiniz; parolayı kopyalayın ve güvenli bir yerde saklayın.

> Önemli: Parola deşifreleme işlemi için VM oluşturulurken iliştirdiğiniz **RSA (PEM)** tipindeki private key gereklidir.

![Get Windows Password diyaloğu](./images/windows/get-windows-password.png)

---

## 8) RDP ile Bağlanma

- Bir RDP istemcisi (ör. Windows Remote Desktop / mstsc) kullanarak Elastic IP veya network içi IP adresine bağlanın.
- Kullanıcı adı: **Administrator**
- Parola: **Get Password** ile elde ettiğiniz parola.

> Güvenlik önerisi: İlk oturumda güçlü bir parola belirleyin ve RDP erişimini yalnızca gerekli kaynaklardan (security group) açın.

---

## Kısa Özet / İpuçları

- **En az 1 NIC** tanımlayın — aksi takdirde VM’in ağ erişimi olmaz.
- **VPC ve subnet** seçimleri genelde varsayılan bırakılabilir; özel gereksinim varsa uygun VPC/subnet’i seçin.
- **Static IP** gerekli değilse otomatik IP (DHCP) seçeneği yeterlidir.
- Aynı ağ içi keşif için **DNS Name** girilmesini öneririz.
- **Cloud-init / ileri ayarlar** bu dokümanda ele alınmaz — ihtiyaç varsa ayrı bir rehbere bakın.
- **Power Up** seçeneği işaretliyse VM sihirbaz sonunda otomatik başlar; aksi halde manuel Start gerekir.
